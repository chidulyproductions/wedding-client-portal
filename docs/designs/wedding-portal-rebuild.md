## Vision

### 10x Check
The 10x version isn't just a music selection tool — it's the full wedding music experience platform. Couples don't just paste URLs; they explore, discover, share, and print their wedding music program. The DJ doesn't just manage clients; they see real-time progress, get notified on updates, and export directly to Spotify for event prep. Every touchpoint feels premium — like the couples hired a tech-forward DJ who invested in their experience.

## Accepted Scope (added to this plan)
- Shareable read-only wedding program page at /program/[client-key]
- Progress indicator (completion ring/bar) on selection page + completion % in admin table
- Dynamic OG meta tags for branded link previews on all client-facing pages
- Admin activity feed with "last activity" timestamps and per-client activity log
- Dark mode toggle on client selection portal
- PDF export button on shareable program page
- Real-time Supabase subscriptions for live multi-user updates
- Rename spotify_url column to media_url
- Email notifications when clients update selections (Supabase webhook → email service)
- Spotify folder export (OAuth + Spotify Web API)
- Responsive/mobile-first design moved to Phase 2 (not Phase 4)
- All 5 error handling requirements (save failure toasts, load error states, oEmbed proxy, atomic deletes, resend button)

## Key Decisions
- Approach: Next.js 14 App Router + TypeScript + Tailwind + Supabase + Vercel
- Supabase: Connect to existing project (no migration needed)
- Auth model: Link-only access for clients (conscious tradeoff for convenience)
- Service role key: Server-only (never in NEXT_PUBLIC_* or client code)
- Responsive: Mobile-first from Phase 2 (not deferred to Phase 4)
- Error handling: Visible failures with retry for all save/load operations
- Mode: SELECTIVE EXPANSION

## Architecture Notes
- Brochure component has 3 render modes (interactive, read-only shareable, PDF) — one component, not three
- oEmbed proxy API route needed for SoundCloud CORS
- Delete operations need atomicity (Supabase RPC or rollback logic)
- Client key derivation must exactly match original regex
